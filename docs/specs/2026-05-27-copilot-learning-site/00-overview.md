# Spec 00 — Vue d'ensemble du site d'apprentissage GitHub Copilot (FR)

**Date** : 2026-05-27
**Statut** : Draft

---

## 1. But

Construire un site Docusaurus **en français** qui apprend aux développeurs à exploiter GitHub Copilot via ses primitives modernes : Instructions, Skills, Agents, APM, MCP, méthodologie de test, Workflows, Copilot CLI, et hygiène token / anti-hallucination.

Le site cible un parcours **~13 h 30** structuré en modules numérotés avec budget temps, complexité étoilée et exercices validants.

## 2. Principe d'autonomie

Cette spec et le plan associé sont **autosuffisants**. Aucune référence externe (site, repo tiers) n'est requise pour reconstruire le site à l'identique. Les sources d'inspiration sont creditées en §6 mais leur disparition ne casse rien.

Tout gabarit, convention, exemple est embarqué dans les specs `01` à `11`.

## 3. Audience cible

| Persona | Niveau | Objectif principal |
|---|---|---|
| Dev débutant Copilot | ⭐ | Comprendre Instructions/Skills/Agents au quotidien |
| Tech lead / référent | ⭐⭐ | Standardiser une équipe via APM, partager des primitives |
| Skill creator rigoureux | ⭐⭐⭐ | Tester, évaluer, designer des workflows custom |

## 4. Livrable

Un patch mergeable dans un Docusaurus existant :
- `docs/**/*.md` (markdown standard, FR)
- `sidebars.ts.fragment.js` (fragment à merger manuellement)
- `.github/agents/*.agent.md` (les 4 agents — voir §5)
- `apm.yml` enrichi pour publier les agents

Pas de modification de `docusaurus.config.ts`.
